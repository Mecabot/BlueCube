Deployment

Pre Deployment Checklist

Ensure that the environment has been setup as per the guide on the Development Environment page. Pay particular attention to ensuring that the three required Arduino libraries have been installed. They are:-

  • Cube4
  • Cube4_ARUtil
  • Adafruit_BluefruitLE_nRF51

Additionally ensure that Ionic, Cordova and at least one of the iOS or Android environments have also been setup.

Change Project ID

The project has a given ID of "com.example.BlueCube". This ID needs to be unique for each developer that builds the apps. This change needs to be completed every time you download a new version of BlueCube (the same unique ID can be used for each version).

To edit it:-

OS X

  1. Browse to "BlueCube" "IonicApp"
  2. Edit config.xml in the text editor of your choise
  3. On the second line where it says id=com.example.BlueCube change it so that it is unique to you. Normally the name ID is in reverse DNS format, so if you own "fredsautos.com", the ID would be id=com.fredsautos.BlueCube. Otherwise just add a random number to the end of "BlueCube", without any spaces or punctuation. If in doubt, add the machines MAC address at the end. On OS X run ifconfig en0 ether | grep ether | awk '{ print $2 }' | sed s/\://g.
  4. Start Terminal "Applications" "Utilities" "Terminal"
  5. Change to the BlueCube "BlueCube" "IonicApp" by cd and dragging the "IonicApp" folder onto the terminal window, then hitting "Enter". For example, I have the BlueCube folder on my desktop, and my username is adam. In this case the command was cd /Users/adam/Desktop/BlueCube/IonicApp
  6. Remove the previously configured platforms a. ionic platform rm ios b. ionic platform rm android
  7. Re-add the iOS platform if you wish to build the iOS app a. ionic platform add ios
  8. Re-add the Android platform if you wish to build the Android app a. ionic platform add android

Windows

  1. Browse to "BlueCube" "IonicApp"
  2. Edit config.xml in the text editor of your choise
  3. On the second line where it says id=com.example.BlueCube change it so that it is unique to you. Normally the name ID is in reverse DNS format, so if you own "fredsautos.com", the ID would be id=com.fredsautos.BlueCube. Otherwise just add a random number to the end of "BlueCube", without any spaces or punctuation. If in doubt, add the machines MAC address at the end. Run getmac, from the command prompt and pick the "Physical Address", removing the hyphens.
  4. Start Command Prompt "Windows Key + R" "cmd.exe"
  5. Change to the BlueCube "BlueCube" "IonicApp" by cd and dragging the "IonicApp" folder onto the command prompt, then hitting "Enter". For example, I have the BlueCube folder on my desktop, and my username is adam. In this case the command was cd "C:\Users\adam\Desktop\BlueCube\IonicApp"
  6. Remove the previously configured platforms a. ionic platform rm ios b. ionic platform rm android
  7. Re-add the Android platform a. ionic platform add android

Build

Every time you modify the BlueCube's code, you need to instruct Ionic to build it. This prepares all of the items in a platform specific way. You build the code from the command line.

OS X

  • Terminal
  • CD
  • ionic build ios (if building iOS app)
  • ionic build android (if building Android app)

Windows

  • CMD.exe
  • CD
  • * ionic build android

Deploy to Mobile Device

Ensure that before deploying to a mobile device, that the latest version of the code has been build, as outlined in the instructions above.

iOS

  1. Connect iPhone, iPad, or iPod Touch to your Mac with a lightning cable
  2. Launch Xcode
  3. Open Project
  4. Unlock Phone
  5. Build
  6. XXX Apple ID stuff XXX
  7. Test App
  8. Stop App via Xcode

Android

  1. Enable Debug mode on phone
  2. Enable USB debug
  3. Launch Android Studio
  4. Open Project
  5. Unlock Phone
  6. Build
  7. Test App
  8. Stop App via Android Studio

At this stage you can quit the IDE, and use the app like a normal app.
